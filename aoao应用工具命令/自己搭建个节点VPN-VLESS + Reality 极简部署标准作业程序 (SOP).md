# 🚀 VLESS + Reality 极简部署标准作业程序 (SOP)

**适用场景**：在纯净 Linux 云服务器上从零搭建代理节点，并使用安卓端直连。
**核心组件**：3x-ui (服务端) + VLESS/Reality 协议 + v2rayNG (客户端)

---

## 🟢 阶段一：服务器基础环境准备与面板安装

**目标：成功安装 3x-ui 并在浏览器中看到登录界面。**

**步骤 1.1：SSH 登录服务器**
1. 打开你的电脑终端（推荐配置为你习惯的绿字黑底极客配色，护眼且专注）。
2. 使用 `ssh root@你的服务器公网IP` 登录服务器。

**步骤 1.2：执行一键安装脚本**
1. 在终端中复制并按回车运行以下命令：
   ```bash
   bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
   ```
2. 脚本运行中途会暂停并询问你是否确认安装，输入 `y` 回车。
3. **设置账户信息（请务必记录）**：
   * `Username` (登录账号): 输入自定义账号并回车。
   * `Password` (登录密码): 输入自定义密码并回车。
   * `Port` (面板访问端口): 输入一个 10000 - 60000 之间的数字（**强烈建议记下这个数字，例如 `54321`**）。

> 🛑 **【Checkpoint 1：安全组放行面板端口】（极其重要，否则绝对卡住）**
> 1. 登录微云（或其他云服务商）的网页控制台。
> 2. 找到该服务器的 **“安全组”** 或 **“防火墙”** 设置。
> 3. 添加一条入站规则：**协议选择 TCP，端口填写你刚才设置的面板端口（如 `54321`），源地址填 `0.0.0.0/0`（允许所有）。**
> 4. 保存生效。

**步骤 1.3：验证面板登录**
1. 打开电脑浏览器，在地址栏输入：`http://你的服务器公网IP:54321` （替换为你实际的 IP 和端口）。
2. 输入刚才设置的账号密码。成功进入后台即代表本阶段 100% 完成。

---

## 🟡 阶段二：配置 VLESS+Reality 节点（防坑核心区）

**目标：创建一个带数字加密签名且不被墙识别的安全节点。**

**步骤 2.1：新建入站规则**
1. 在 3x-ui 面板左侧菜单点击 **Inbounds (入站列表)**。
2. 点击绿色的 **+ Add Inbound (添加入站)** 按钮。

**步骤 2.2：填写节点参数（请严格对照）**
* **Remark (备注)**：随便填（如 `HK-Node`）。
* **Protocol (协议)**：选择 `vless`。
* **Port (端口)**：随意输入一个未使用的端口（**务必记下，例如 `57777`**）。
* **Network (网络)**：选择 `tcp`。
* **Security (安全性)**：选择 `reality`。
* **uTLS**：保持默认 `chrome` 即可。

**步骤 2.3：设置伪装目标 (Dest)**
向下滚动找到 `Reality` 设置区：
* **Dest (目标网站)**：填入 `www.microsoft.com:443`
* **Server Names (SNI)**：填入 `www.microsoft.com`

**步骤 2.4：生成专属密钥（新手必死坑位，必须点！）**
1. 在 Reality 设置区，找到 `Private Key` (私钥) 和 `Public Key` (公钥) 的输入框。它们此时是空的。
2. **点击输入框旁边的 🔄 (获取新密钥 / Get new keys) 按钮。**
3. 确认框内自动填满了长串的随机字符。
4. 点击最下方的 **Add (添加)** 按钮保存。

> 🛑 **【Checkpoint 2：重启核心与放行节点端口】（极其重要）**
> 1. 回到面板左侧菜单的 **Overview (系统状态)**。
> 2. 找到 `Xray` 卡片，点击 **Restart (重启)**。等待 3 秒，确保 Xray 状态显示为绿色的 **Active / Running**。
> 3. **立刻去云服务商的网页控制台**，再次进入安全组/防火墙。
> 4. 添加一条入站规则：**协议 TCP，端口填写你刚才新建的节点端口（如 `57777`），源地址 `0.0.0.0/0`。**保存。

## 🔵 阶段三：手机端一键直连

**目标：手机成功解析节点配置并实现科学上网。**

**步骤 3.1：获取节点二维码**
1. 回到电脑上的 3x-ui 面板，进入 **Inbounds (入站列表)**。
2. 找到建好的节点，点击列表最左侧微小的 **`+` 号** 展开详情。
3. 点击底部的 **二维码图标 (QR Code)**，屏幕上会出现一个巨大的二维码。

**步骤 3.2：下载正确架构的客户端**
1. 在你的 Galaxy Z Flip 4 上，打开浏览器访问 v2rayNG 的官方发布页：
   `https://github.com/2dust/v2rayNG/releases`
2. 认准带有 `Latest` 标签的版本，展开 `Assets`。
3. **点击下载 `v2rayNG_xxx_arm64-v8a.apk`** （专为该机型的骁龙处理器优化的 64 位版本，运行最稳定）。
4. 安装并打开 App。

**步骤 3.3：扫码连接**
1. 在 v2rayNG 主界面，点击右上角的 **`+` 号**。
2. 选择 **【扫描二维码】**。
3. 对准电脑屏幕上的二维码扫描。节点会瞬间出现在手机列表里。
4. 点击该节点（左侧出现竖线代表已选中）。
5. 点击右下角的 **V 字圆形按钮** 启动连接（首次连接手机系统会弹出 VPN 授权提示，点击允许）。

> 🛑 **【Checkpoint 3：最终连通性验证】**
> 1. 连接启动后，点击 v2rayNG 底部的 **“测试全部配置真连接”**（或者直接点击节点的延迟数字区）。
> 2. 如果显示具体的延迟数字（例如 `150ms`），则说明大功告成！
> 3. 打开手机上的浏览器或 Twitter App 验证网页是否能正常加载。

### 📝 常见故障速查表（遇到问题不要慌，查表即解）

| 故障现象                                     | 根本原因                     | 唯一解决动作                                                 |
| :------------------------------------------- | :--------------------------- | :----------------------------------------------------------- |
| **浏览器打不开 3x-ui 面板，一直转圈**        | 服务器面板端口未开放         | 去云服务商控制台安全组，**放行面板端口** (如 54321)。        |
| **面板内 Xray 状态显示红色 Error**           | 创建节点时忘记生成密钥       | 去入站列表编辑节点，在 Reality 区**点击生成密钥**并保存，再重启 Xray。 |
| **手机测速显示 `context deadline exceeded`** | 服务器节点端口未开放         | 去云服务商控制台安全组，**放行节点端口** (如 57777)。        |
| **手机软件报 `yaml: unmarshal error`**       | 客户端软件老旧，不支持新协议 | 卸载原版 Clash，严格按照 3.2 步骤**下载 v2rayNG**。          |
